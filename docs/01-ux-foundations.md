# UX Foundations

## Project Name

**NoteFlow**

## Project Type

Mobile UX/UI case study for a note-reading practice app.

---

## What is NoteFlow?

NoteFlow is a mobile app concept designed for beginner music students who want to improve their note-reading skills through short, simple, and motivating practice sessions.

The app helps users practice recognizing notes on the music staff, receive instant feedback, review weak notes, and track their progress over time.

---

## User

The main target user is a beginner music student.

This user may be:

* A beginner piano student
* A student learning basic music theory
* Someone who struggles with recognizing notes quickly
* Someone who wants short daily practice instead of long and stressful exercises

---

## User Problem

Beginner music students often find note reading difficult at first.

They may:

* Recognize notes slowly
* Confuse some notes on the staff
* Feel bored by traditional exercises
* Feel stressed when they make mistakes
* Lose motivation if they cannot see their progress

---

## User Needs

The user needs a practice experience that is:

* Simple
* Short
* Clear
* Beginner-friendly
* Motivating
* Easy to use on mobile
* Supportive after mistakes

---

## Product Goal

The goal of NoteFlow is to help beginner music students improve note recognition through:

* Short note-reading exercises
* Multiple-choice questions
* Instant feedback
* Weak note review
* Progress tracking
* A calm and friendly interface

---

## Main User Flow

The main flow of the app is:

**Welcome → Home → Practice → Result → Progress**

### Flow Explanation

1. The user opens the app and sees the Welcome Screen.
2. The user enters the Home Screen.
3. The user starts a short note-reading practice.
4. The user answers multiple-choice note questions.
5. The user sees the result and accuracy.
6. The user can review weak notes or check progress.

---

## UX Goals

The main UX goals of NoteFlow are:

### 1. Make practice easy to start

The user should quickly understand where to begin.

### 2. Reduce stress

The interface should feel calm and supportive, not like a difficult exam.

### 3. Give clear feedback

The user should immediately understand whether an answer is correct or wrong.

### 4. Support motivation

Progress tracking, streaks, and weak notes should help users continue practicing.

### 5. Keep the experience simple

Since the target user is a beginner, the app should avoid unnecessary complexity.

---

## Key Design Decisions

### Short Practice

The app focuses on short practice sessions because beginners may not want long exercises.

### Instant Feedback

The practice screen gives quick feedback so users can learn from their answers.

### Weak Notes

The app shows weak notes to help users understand what they need to review.

### Progress Tracking

The progress screen helps users see improvement over time.

### Calm Visual Style

The visual design uses soft colors, rounded cards, and a clean layout to create a friendly learning experience.

---

## Summary

NoteFlow is designed around a simple UX foundation:

A beginner music student needs a short, clear, and motivating way to practice note reading.

The product focuses on helping the user start practice easily, receive feedback, understand mistakes, and track progress in a calm mobile experience.
