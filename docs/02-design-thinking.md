# Design Thinking

## Overview

Design Thinking is a problem-solving process used to understand users, define their problems, create ideas, build prototypes, and test solutions.

For NoteFlow, I used Design Thinking to turn a simple idea into a more structured UX/UI project.

---

## 1. Empathize

In this step, the goal is to understand the user and their experience.

The target user of NoteFlow is a beginner music student who is learning note reading.

This user may feel:

* Confused when reading notes on the staff
* Slow when trying to recognize notes
* Stressed after making mistakes
* Bored by traditional exercises
* Unmotivated if progress is not visible

### Empathy Statement

Beginner music students need a simple and supportive way to practice note reading because traditional exercises can feel confusing, repetitive, or stressful.

---

## 2. Define

In this step, the main problem is clearly defined.

### Problem Statement

Beginner music students often struggle to recognize notes quickly on the staff, and traditional practice methods may feel boring or stressful.

### How Might We Question

How might we help beginner music students practice note reading in a short, simple, and motivating way?

---

## 3. Ideate

In this step, possible solutions are explored.

For NoteFlow, the main ideas were:

* Short daily practice sessions
* Multiple-choice note recognition
* Instant feedback after each answer
* Result screen after practice
* Weak note review
* Progress tracking
* Weekly accuracy chart
* Practice streak
* Calm and beginner-friendly visual style

These ideas were selected because they directly support the user’s need for simple, clear, and motivating practice.

---

## 4. Prototype

In this step, the idea was turned into a clickable mobile prototype in Figma.

The prototype includes five main screens:

1. Welcome Screen
2. Home Screen
3. Practice Screen
4. Result Screen
5. Progress Screen

### Prototype Flow

Welcome → Home → Practice → Result → Progress

The prototype helps show how the user moves through the main experience of the app.

---

## 5. Test

This project is currently a portfolio prototype, so full usability testing has not been completed yet.

In a future version, I would test the prototype with 2–3 beginner music students.

### Test Goals

I would check whether users can:

* Understand the purpose of the app
* Find the Start Practice button easily
* Answer a note question without confusion
* Understand correct and wrong feedback
* Understand the result screen
* Find and understand the progress screen

### Possible Improvements After Testing

Based on user feedback, I would improve:

* Screen clarity
* Button labels
* Feedback messages
* Practice flow
* Accessibility
* Progress visualization

---

## Design Thinking Summary

The Design Thinking process helped structure the NoteFlow project from user problem to prototype.

The main idea was not only to create a beautiful UI, but to design a simple learning experience for beginner music students.

The final concept focuses on:

* Understanding beginner users
* Reducing stress
* Making practice short and clear
* Giving instant feedback
* Showing progress over time
